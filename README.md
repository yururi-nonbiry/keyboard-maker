# Keyboard Maker (仮)

自作キーボードを設計するためのフロントエンド主導型 Web アプリケーションです。

## 開発環境の起動

本プロジェクトは Docker Compose を使用して開発環境を構築できます。

```bash
docker compose up
```

起動後、[http://localhost:5173](http://localhost:5173) にアクセスすることで開発サーバーを利用できます。

---

## 仕様書

### 1. コンセプト
本プロジェクトは、ウェブブラウザ上で動作する「フロントエンド主導型」の自作キーボード設計ツールです。
ユーザーは外部にデータを送ることなく、ブラウザ内でジオメトリ計算、プレビュー、および設計ファイルの生成を完結させることができ、プライバシーと自由度の両立を目指します。

### 2. システム構成
#### フロントエンド (重厚なCADツール)
- **Framework**: React + TypeScript
- **Build Tool**: Vite
- **3D Rendering**: Three.js / React Three Fiber
- **State Management**: Zustand (設計データの管理)
- **Styling**: Vanilla CSS (CSS Modules)
- **Storage**: File System Access API (ローカルファイルへの直接保存), IndexedDB (一時保存)

#### バックエンド (軽量マイクロサービス)
- **Platform**: AWS Lambda
- **用途**: フロントエンドでは困難な重い計算やフォーマット変換
    - PCBの自動ルーティング (KiCad Python API 等)
    - 複雑な3Dモデルの生成 (CadQuery / OpenCASCADE による STEP 出力)
    - ファームウェアビルド (QMK/ZMK 環境の提供)

### 3. 主要機能
#### 3.1 レイアウト編集
- Keyboard Layout Editor (KLE) 形式のインポート/エクスポート
- キースイッチの動的配置 (座標、回転角)
- スタビライザーの自動配置オプション
- **物理干渉チェック**: キーキャップ同士の衝突やピッチ不足をリアルタイムで警告

#### 3.2 スイッチ・パーツ選択
- 主要スイッチ規格のサポート (MX互換, Choc V1/V2, X Switch, 静電容量無接点等)
- 各キーごとの個別設定 (スイッチ種別、ハンダ穴 vs ソケット、LED の向き)
- **コントローラー・テンプレート**: XIAO nRF52840 や Pro Micro 等の配置、およびピン配置の事前定義

#### 3.3 リアルタイム・プレビュー
- 2D パターンプレビュー (SVG) - 基板のシルクや外形の確認
- 3D ジオメトリプレビュー (Three.js/R3F) - 筐体を含めた立体的な確認

#### 3.4 ファイル出力
- **KiCad S式エクスポート**: .kicad_pcb および .net ネットリストの生成（KiCad側での配線作業を前提とした最適化）
- **高精度3Dモデル**: アッセンブリ構造を維持した STEP ファイル、OpenSCAD ソースコード
- **マニュアル加工用データ**: レーザーカット用 DXF、等身大確認用 PDF/SVG

### 4. 詳細技術仕様 (Maniac/Professional Requirements)
ユーザーが KiCad や 3D CAD にインポートした後、マニアックなカスタマイズを継続できるための「データ品質」にこだわります。

- **KiCad 連携の深化**: 
    - S式出力時、各フットプリントを `(group ...)` で括り、KiCad 上での一括移動や選択を容易にします。
    - ユーザー指定のフットプリントライブラリエイリアスに対応。
- **STEP ファイルの構造化**: 
    - スイッチ、プレート、ケースを個別のエンティティとして出力。
    - ネジ穴の中心線データを出力し、CAD 上での位置合わせを容易にします。
- **パラメトリック設計**: 
    - 外形線の角丸、スルーホール径、プレートの厚みなどをパラメータ化し、出力前にユーザーが微調整可能。

### 5. データ構造 (JSON Schema)
プロジェクトファイルは以下の情報を包含する独自形式 (JSON) とします。
- `metadata`: 作者、バージョン等の基本情報
- `layout`: 各キーの座標、角度、スイッチ種別、キーキャップサイズ
- `pcb_config`: コントローラー位置、ダイオードの向き、フットプリント属性
- `case_config`: ケース外形、ネジ穴位置、壁の厚み等のパラメータ

### 6. 開発環境
- **Docker Compose**: フロントエンド開発環境をコンテナで管理
- **Hot Module Replacement (HMR)**: Vite による高速な開発サイクル
- **技術スタック**: React 18 / TypeScript / Three.js / zustand

### 7. 今後の拡張性
- ユーザーによるカスタムフットプリント (.kicad_mod) のアップロード機能
- ブラウザ上での簡易的な回路図エディタとの連動
- ZMK / QMK ファームウェアの keymap.conf 自動生成
